# Obsidian Gantt 测试文档

## 1. 数据连接器 (Data Connectors)

### 1.1 CSV 连接器
- [x] **正常加载** — CSV 文件包含有效的 persons、projects、tasks 数据，验证图表正确渲染
- [ ] **空 CSV 文件** — 仅有表头无数据行，验证不崩溃且显示空状态
- [x] **仅表头 CSV** — 无数据行，验证返回空数组无报错
- [ ] **CSV 文件缺失** — 删除或移动 CSV 文件后刷新，验证显示明确错误信息
- [ ] **BOM 头处理** — CSV 包含 UTF-8 BOM (`﻿`)，验证首列名不被破坏
- [ ] **CRLF / 混合换行** — 文件使用 `\r\n` 或混合换行符，验证正确解析
- [ ] **引号内嵌换行** — CSV 字段引号内包含换行符，验证行分割正确
- [ ] **连续空行** — 文件末尾有连续空行，验证不产生空数据行
- [ ] **自定义分隔符** — 使用 Tab 或分号作为分隔符，验证 `CsvParseOptions.delimiter` 生效
- [ ] **自定义列映射** — 在 connector 配置中自定义列名映射，验证字段正确对应
- [ ] **包含 metadata 字段** — CSV 含额外未知列，验证存入 `metadata` 且不影响核心功能

### 1.2 多连接器视图
- [ ] **合并多个连接器** — 视图引用多个 connector ID，验证数据正确合并去重
- [ ] **同名人员/项目冲突** — 多个连接器包含同名 ID 的人员或项目，验证合并策略
- [ ] **某连接器加载失败** — 多连接器视图中一个加载失败，验证其他连接器数据正常显示

### 1.3 连接器脚本
- [ ] **脚本文件缺失** — connector 引用的 JS 文件不存在，验证报 `CONNECTOR_NOT_FOUND`
- [ ] **缺少 fetch 导出** — 脚本未导出 `fetch` 函数，验证报 `CONNECTOR_SCRIPT_ERROR`
- [ ] **缺少 transform 导出** — 脚本未导出 `transform` 函数，验证报错
- [ ] **fetch 返回异常数据** — `fetch()` 抛出异常或返回非预期格式，验证错误处理
- [ ] **transform 返回异常数据** — `transform()` 抛出异常，验证错误捕获和提示
- [ ] **push 不支持** — connector 无 `push` 导出，验证推送时提示 "does not support push"

### 1.4 数据刷新
- [ ] **手动刷新** — 修改上游数据后触发刷新，验证缓存更新且本地编辑保留
- [ ] **刷新失败** — 网络/Vault 读取失败，验证错误信息写入 `store.error`
- [ ] **刷新后合并** — 上游数据变更后刷新，验证 merge engine 正确合并本地编辑

## 16. 边界和异常情况

### 16.1 空数据
- [ ] **无视图** — "No view selected" 空状态显示
- [ ] **视图无连接器** — 视图有定义但 connectors 列表为空
- [ ] **无人员** — 无 person 数据，人员面板显示正常无崩溃
- [ ] **无项目** — 无 project 数据，项目面板显示正常无崩溃
- [ ] **无任务** — 无 task 数据，时间轴范围显示今日
- [ ] **项目无任务** — 项目存在但无关联任务，显示在 Unassigned 面板
- [ ] **人员无任务** — 人员存在但无关联任务

### 16.2 加载状态
- [ ] **加载中** — 数据加载时显示 "Loading..." 居中提示
- [ ] **加载错误** — 数据加载失败显示红色错误信息
- [ ] **加载后正常** — 加载完成后 Loading 状态消失

### 16.3 极端数据
- [ ] **超长任务名** — 任务标题超过 100 字符，任务条内截断
- [ ] **超长项目名** — 项目名称过长，TaskList 侧栏截断
- [ ] **大量任务 (>500)** — 性能可接受，滚动不卡顿
- [ ] **大量项目 (>100)** — 性能可接受
- [ ] **大量人员 (>50)** — 性能可接受
- [ ] **超长时间跨度 (>10年)** — 任务跨越多年，头部和网格正确
- [ ] **历史日期 (1970前)** — 极早的日期正确渲染
- [ ] **未来日期 (2100后)** — 极远的日期正确渲染
- [ ] **同日多任务** — 同一人员同一天有多个任务，lane 分配正确
- [ ] **全重叠任务** — 所有任务完全同时间段，lane 不重叠

### 16.4 异常输入
- [ ] **非法日期字符串** — `startDate` 为 "invalid-date"，验证不崩溃
- [ ] **progress 越界** — `progress` 为 -1、101、NaN
- [ ] **空字符串 ID** — task/project/person 的 ID 为空字符串
- [ ] **null/undefined 字段** — 各字段为 null 或 undefined 时
- [ ] **特别字符** — 名称中包含 `<script>`、`\n`、emoji、Unicode 等特殊字符
- [ ] **循环依赖** — 任务 A 依赖 B，B 依赖 A
- [ ] **不存在的人员引用** — task 的 `personId` 指向不存在的人员
- [ ] **不存在的项目引用** — task 的 `projectId` 指向不存在的项目

### 16.5 并发和时序
- [ ] **快速连续拖拽** — 快速拖拽多个任务条，操作不互相干扰
- [ ] **拖拽中刷新** — 拖拽过程中触发数据刷新（应阻止或正确处理）
- [ ] **快速切换视图** — 连续切换视图，前一个视图的异步操作不影响当前
- [ ] **推送中编辑** — 推送未完成时进行编辑操作
- [ ] **同时多人编辑** — （如适用）两人同时编辑同一视图

### 16.6 Obsidian 特性
- [ ] **深色/浅色主题** — 切换 Obsidian 主题，样式使用 CSS 变量正确适配
- [ ] **窗口大小变化** — 调整 Obsidian 窗口大小，图表自适应
- [ ] **侧栏收起/展开** — Obsidian 侧栏状态变化时布局正确
- [ ] **Vault 切换** — （如适用）切换 Vault 后数据隔离
- [ ] **插件禁用/启用** — 禁用后重新启用插件，数据完整
- [ ] **Obsidian 移动端** — （如适用）在手机/平板上操作
